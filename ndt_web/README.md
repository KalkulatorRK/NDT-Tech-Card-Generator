# НК-Карта — Автоматизация неразрушающего контроля

Веб-приложение для автоматизации разработки технологических карт неразрушающего контроля (НК) и оценки качества сварных соединений в области атомной энергетики.

## Возможности

- **Генерация технологических карт** радиографического контроля по ГОСТ Р 50.05.07-2018
- **Оценка качества** сварных соединений по НП-105-18
- Аутентификация и личный кабинет
- Роли пользователей (администратор / пользователь / гость)
- Интеграция с платёжной системой ЮKassa
- Скачивание карт в форматах DOCX и PDF
- Административная панель Django

|----------|----------|--------|

## Технологии

- **Backend:** Python 3.12 + Django 4.2
- **База данных:** SQLite (разработка) / PostgreSQL (продакшн)
- **Генерация документов:** python-docx (DOCX), reportlab (PDF)
- **Frontend:** Bootstrap 5 + Vanilla JS
- **Платежи:** ЮKassa (Яндекс Касса)

## Установка и запуск

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка окружения

```bash
cp .env.example .env
# Отредактируйте .env, установив SECRET_KEY и другие параметры
```

### 3. Инициализация базы данных

```bash
python manage.py migrate
python manage.py init_data          # Загрузка начальных данных
python manage.py collectstatic
```

### 4. Запуск сервера разработки

```bash
python manage.py runserver
```

Приложение будет доступно по адресу: http://localhost:8000

### 5. Вход в административную панель

URL: http://localhost:8000/admin/  
Логин: `admin` / Пароль: `admin`

> ⚠️ Измените пароль администратора после первого входа!

## Настройка ЮKassa

В файле `.env` установите:

```
YOOKASSA_SHOP_ID=ваш_shop_id
YOOKASSA_SECRET_KEY=ваш_secret_key
SITE_URL=https://ваш-домен.ru
```

При отсутствии настроек ЮKassa платежи работают в тестовом режиме
(автоматическое подтверждение без реальной оплаты).

## Безопасность

При переводе в продакшн обязательно:
1. Установите сложный `SECRET_KEY` в `.env`
2. Установите `DEBUG=False`
3. Настройте HTTPS
4. Укажите корректные `ALLOWED_HOSTS`
